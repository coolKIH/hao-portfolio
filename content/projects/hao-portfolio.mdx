This project is more than a simple showcase; it’s a technical sandbox where I iterate on peak performance and minimalist aesthetics.

**Hao's Portfolio** is built on the **Next.js App Router** architecture, with core logic driven by **Server Components**. By using **MDX** as a local-first content engine, I’ve balanced developer flexibility with low-maintenance stability. Through a library of reusable **React** primitives, the storytelling in both the Blog and Projects sections is both fluid and structurally sound.

### Core Features

- **Content-First Architecture**: By colocating blogs and projects as local **MDX** files, I’ve eliminated external CMS overhead. This ensures full version control and a frictionless, offline-capable writing workflow.
- **Fluid Responsiveness**: Engineered with **Tailwind CSS 4**, the UI provides a consistent, high-fidelity experience across everything from mobile touchpoints to ultra-wide desktop displays.
- **Adaptive Theme Engine**: Leveraging **next-themes**, the site intelligently mirrors system preferences, ensuring a comfortable reading experience that respects the user's environment.
- **Inclusive & Accessible Design**: Prioritizing high-contrast typography and semantic HTML (A11y) to ensure a premium, barrier-free reading experience for all users.
- **SEO & Core Web Vitals**: Utilizing **Static Site Generation (SSG)** and **React Server Components (RSC)**, the site delivers a near-instant Time to First Byte (TTFB) while ensuring search engines can easily index its content.

### Technical Highlights

- **Lean Rendering Strategy**: By maximizing **RSCs**, I’ve moved the heavy lifting to the server, significantly reducing the client-side **JavaScript footprint**. 
- **Advanced MDX Image Engine**: Custom-engineered MDX components that map standard Markdown syntax to `next/image`. This system handles automatic format conversion (WebP/AVIF) and layout-shift prevention, while adding support for **optional semantic captions**—providing a polished, magazine-like reading experience.
- **Atomic Design System**: A unified styling layer built on **Tailwind** and **shadcn** primitives, providing a predictable, accessible, and scalable design language that remains easy to refactor.
- **Rich-Media MDX**: Utilizing **next-mdx-remote** with RSC support, I can embed interactive modules and rich media directly within articles, bypassing the expressive limits of standard Markdown.
- **End-to-End Type Safety**: The entire stack is hardened with **TypeScript**. From data fetching to UI rendering, strict typing minimizes runtime regressions and improves long-term maintainability.
- **GitOps Workflow**: Content and code live in a single source of truth. This approach simplifies the pipeline from a local `commit` to a global Vercel deployment.
