This project serves as both a personal showcase and a technical sandbox for exploring minimalist web patterns and performance optimization.

**Hao's Portfolio** is built on the **Next.js App Router**, with logic primarily handled by **Server Components**. By using **MDX** as a local-first content engine, the site achieves a balance between writing flexibility and architectural stability. A library of shared **React** primitives ensures that both the Blog and Projects sections remain visually and structurally consistent.

### Core Features

- **Content-First Architecture**: Blogs and projects are managed as local **MDX** files, removing the need for an external CMS. This approach provides full version control and a straightforward, offline-ready workflow.
- **Fluid Responsiveness**: Built with **Tailwind CSS 4**, the layout adapts across mobile, tablet, and desktop viewports while maintaining a consistent design system.
- **Adaptive Theme Engine**: Implemented via **next-themes**, the UI respects system color preferences to provide a comfortable reading experience in both light and dark modes.
- **Semantic Accessibility**: Focused on high-contrast typography and semantic HTML to ensure the site is navigable and readable for all users.
- **SEO & Performance**: Leverages **Static Site Generation (SSG)** and **React Server Components** to minimize initial load times and ensure content is easily indexable by search engines.

### Technical Highlights

- **Lean Client Footprint**: By utilizing **RSCs**, the application minimizes the amount of client-side JavaScript required, leading to faster interaction times.
- **Custom MDX Image Engine**: Custom components map standard Markdown image syntax to `next/image`. This handles automatic format optimization (WebP/AVIF) and prevents layout shifts, with support for semantic captions.
- **Atomic Styling**: Uses a unified design layer based on **Tailwind** and **shadcn/ui**, creating a predictable and scalable CSS structure that is easy to maintain.
- **Flexible MDX Rendering**: Uses **next-mdx-remote** to render rich content, allowing for the inclusion of interactive React components directly within Markdown files.
- **Type-Safe Development**: The project is built with strict **TypeScript** configurations. From data retrieval to UI components, typing helps prevent regressions and clarifies the data flow.
- **Integrated Workflow**: Code and content are colocated in a single repository, allowing for a simplified deployment process from a local `git commit` to production.

### Personal Reflection

Building this site was a conscious exercise in **essentialism**. In a web ecosystem often filled with heavy dependencies and tracking scripts, I wanted to create a space that is technically sound yet visually quiet.

The primary challenge was "un-learning"—deciding which features to omit to ensure the content remains the priority. This portfolio reflects my belief that good engineering should be felt rather than seen, manifesting as speed, clarity, and ease of use.
