## 🚀 Portfolio Engineering

### Major Architecture Change
I’ve transitioned from fetching Blog content via the GitHub API to serving it directly from the local `content/` directory.
* **Why:** To remove unnecessary external data requests and simplify the workflow.
* **Benefit:** This refactoring allows the Portfolio to render media resources (images, videos) directly from the `public/` directory, leveraging static resource optimization instead of dynamic API calls.

### Feature Updates
* **Dynamic Pagination:** Migrated from query parameters (`/blog/?page=1`) to dynamic routes (`/blog/page/1`). This enables **Static Site Generation (SSG)** for all paginated lists, making the browsing experience seamless.
* **MDX Image Component:** Added support for a custom `<Image>` component with optional Captions, fully integrated with `next/image` for performance.
* **Documentation:** Updated the project **README** to reflect the latest architecture and instructions.
* **Cleanup:** Removed the legacy "Lab" page that was used for initial testing.

## 🧠 Pickup & Takeaways

### The Mindset Shift
* **"Is it worth doing?" vs. "Can I do it?":** As engineers, we often get caught up in the technical possibility. This week, I’ve started asking about the value first.
* **Feature-thinking:** For any new requirement, I now pause to think: *What am I solving? What are the options A/B/C? What are my concerns?* Then, I use AI to amplify my execution. This leads to more clarity and less friction.
* **Don't Over-engineer:** I gave up the "two-repo architecture" because the "need for private blog posts" was a hypothetical scenario that never materialized. **Less is more.**

### Technical Insights
* **The Power of TypeScript:** TS isn't just about catching errors at compile time. It acts as the **best form of documentation**. Enforcing types makes collaboration smoother and refactoring far less error-prone. It’s about clarity for both the developer and the IDE.
* **Case-insensitive Filesystems:** A reminder for macOS users: APFS ignores case changes in filenames. If you rename `Nav.tsx` to `nav.tsx`, Git might not see it, leading to production build errors. Always use `git mv`.
* **Baseline (MDN):** I’ve gained a new appreciation for this keyword. It provides a clear standard for "widely available" features, helping me decide which modern CSS/JS to adopt with confidence.

### Personal Growth
* **Letting go of "Professional Exposure" fear:** I'm becoming more open to judgment and failure. Being "out there" is the first step toward new opportunities.
