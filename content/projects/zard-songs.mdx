## 1. The Vision: A Legacy Reborn

**Zard Songs** is a digital sanctuary dedicated to **Izumi Sakai (ZARD)**. Originally conceived in 2018 as a 25th-anniversary tribute, the project has evolved from a local data-crawling experiment into a polished, mobile-responsive web application.

* **The Problem**: ZARD’s lyrics are often scattered across the web in fragmented or poorly translated formats.
* **The Solution**: A centralized, "three-dimensional" lyrics engine presenting the Japanese original, Chinese translation, and Romaji side-by-side, paired with integrated audio playback to help fans learn and sing along.
* **The Design Language**: A "Nostalgic-Minimalist" aesthetic, utilizing the iconic purple tones from the *Yureru Omoi* era to evoke the feeling of a live concert.

---

## 2. Core Features (The User Experience)

* **The "Eternal Hero" Section**: A 100vh immersive landing experience featuring a fixed-position tribute photo that remains static as the page content scrolls over it, creating a deep parallax effect.
* **Trilingual Lyric Engine**: A linear, performance-optimized archive of 90 songs. Lyrics are mapped in a 3-column grid for real-time comparative reading.
* **Dynamic Search & Filtering**: A lightning-fast frontend filter that allows users to instantly find tracks by title within the 300KB+ JSON dataset.
* **Integrated Media Player**: A custom-built, floating audio interface that manages playback states without refreshing the page.
* **Smart Pagination**: To ensure fast initial load times, the app uses a "Load More" pattern, rendering only 6 songs by default to keep the DOM lightweight.

## 3. Technical Highlights (The Engineer’s View)

### The Stack
* **Frontend**: Vue.js 2.7 (CDN-loaded) + Vanilla JavaScript.
* **Styling**: Bootstrap 5 + Custom CSS (utilizing modern properties like `clip-path`).
* **Data Architecture**: A structured 306KB JSON payload containing all song metadata and lyrics, fetched asynchronously via `vue-resource`.

### Engineering Challenges & Solutions

#### **The Mobile "Fixed-Background" Paradox**
Replicating the "fixed background" hero effect on mobile browsers (iOS Safari/Chrome) is a notorious challenge due to limited support for `background-attachment: fixed`.
* **The Fix**: I implemented a modern CSS **`clip-path`** strategy. By defining a fixed container and clipping the viewable area, I bypassed mobile hardware-acceleration limitations, achieving a smooth 60fps parallax effect across all devices.

#### **Stateful Audio Management**
Even in a Single Page Application without a complex state manager like Vuex, I maintained a robust audio state.
* **Mechanism**: A hidden HTML5 `<audio>` element serves as the "source of truth." Vue data observers monitor the element's event listeners—specifically `playing`, `ended`, and `error`—to update the global `playStatus` and toggle the "Now Playing" floating UI. This ensures immediate feedback (e.g., "(ERROR Loading)") if a link is broken.

#### **Performance & Asset Loading**
* **Pre-fetching**: To prevent "pop-in" during the hero animation, I implemented a manual image pre-loader in `main.js` using `new Image().src`, ensuring the tribute photo is pre-loaded to prevent visual 'pop-in' during the hero fade-in transition.
* **Asset Decoupling**: To keep the Git repository lean, large media files (MP3s) are hosted externally and linked dynamically, separating application logic from heavy binary assets.
