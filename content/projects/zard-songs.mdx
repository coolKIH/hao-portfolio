## 1. Project Overview

**Zard Songs** is a specialized web application designed to archive and present the lyrics of **Izumi Sakai (ZARD)**. Originally started in 2018 as a data-crawling experiment, the project evolved into a responsive frontend application focused on bilingual accessibility and clean media integration.

* **Objective**: To centralize ZARD’s discography, which is often fragmented across various web sources.
* **The Solution**: A comparative lyrics engine displaying Japanese, Chinese, and Romaji side-by-side, paired with a custom audio interface for synchronized listening.
* **Design Approach**: A minimalist interface utilizing a purple-toned palette inspired by the *Yureru Omoi* album era, prioritizing content readability over decorative elements.

---

## 2. Functional Features

* **Immersive Hero Section**: A full-height landing view utilizing a fixed-position background that stays static during scrolling, providing a clean visual introduction to the site.
* **Trilingual Lyric Layout**: An archive of 90 songs organized in a three-column grid. This layout allows for real-time comparison between the original Japanese text and its translations.
* **Instant Filter System**: A frontend search implementation that filters the 300KB+ JSON dataset in real-time as the user types, providing immediate access to specific tracks.
* **Integrated Audio Controls**: A floating playback interface that manages global audio states, allowing users to listen to tracks while navigating the lyrics archive.
* **Incremental Rendering**: To maintain performance, the application renders 6 songs by default, using a "Load More" pattern to keep the DOM size manageable.

## 3. Technical Implementation

### The Stack
* **Frontend**: Vue.js 2.7 (CDN-loaded) and Vanilla JavaScript.
* **Styling**: Bootstrap 5 and Custom CSS (including `clip-path` for modern layout effects).
* **Data Management**: A structured 306KB JSON payload containing song metadata, fetched asynchronously via `vue-resource`.

### Engineering Challenges

#### **Mobile Parallax Stability**
Implementing a "fixed background" effect on mobile browsers is often inconsistent due to varying support for `background-attachment: fixed`.
* **Solution**: I used a **`clip-path`** strategy. By defining a fixed container and clipping the viewable area, I bypassed hardware-acceleration limitations on iOS and Android browsers, ensuring a smooth scroll effect across devices.

#### **Stateful Audio Logic**
Without a dedicated state management library like Vuex, I implemented a centralized audio controller within the main Vue instance.
* **Mechanism**: A hidden HTML5 `<audio>` element acts as the source of truth. Vue data observers monitor listeners like `playing`, `ended`, and `error` to update the global `playStatus`. This provides immediate UI feedback if a media link fails to load.

#### **Asset Optimization**
* **Image Pre-loading**: To prevent visual "pop-in" during the initial fade-in transition, I implemented a manual pre-loader in `main.js` that triggers before the hero animation begins.
* **External Hosting**: Large MP3 files are hosted on external servers and linked dynamically. This keeps the Git repository lightweight while allowing the application to scale its content without increasing build times.

## 4. Personal Reflection

This project reflects an iterative approach to development. It began as a **Python-based web scraping** project to gather and clean unstructured data from various forums and has since served as a practical study in **UI/UX refinement**.

The primary focus was moving from "raw" data display to an integrated layout where playback and search controls are accessible without obstructing the visual experience. It was a lesson in building a specialized tool that serves a specific community by focusing on clarity, speed, and functional design.
